# OctoAcme Project Management Docs

This folder contains the institutional project management guidance used by OctoAcme teams.

## Brief process overview
OctoAcme manages projects with a lightweight lifecycle focused on customer value, iterative delivery, and clear ownership.

1. Initiation: define the problem, success metrics, stakeholders, and a go/no-go decision.
2. Planning: build a prioritized backlog, define acceptance criteria and Definition of Done, and map milestones and dependencies.
3. Execution and tracking: deliver in small increments, use pull requests and CI checks, and escalate blockers quickly.
4. Release and deployment: follow a release checklist, validate post-deploy behavior, and communicate outcomes.
5. Retrospective and improvement: capture learnings, assign actions, and track improvements in the backlog.

Supporting practices include explicit role definitions, risk register maintenance, and recurring stakeholder communication.
